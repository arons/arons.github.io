Blog redesign

24 Aug 2021

It was a long time ago when I last wrote something on this blog.

Mostly was due to the tedius process to add some new content, but now I ended up having some time to change the structure of the blog.

Of course the blog structure is not the only to blame...it takes me a lot of time to write a post, but to the above problem I can figure out a solution, for this one the only solution is to write more.

I wanted to have the blog as simple as possible, with the possibility to deploy it everywhere I want in case I want to move it.
Also should be easy to add new post or change them.
So I ended up with a simple html static structure. The blog has only two files that serve as entry point: the index.html and view.html.
The posts are organized into a directory structure like year/month/day.
Every post is compose of two files: the post itself and an overview to be used in the main page.

The index.html and view.html shoul not be modified by a new post. The only file to modify is the main.html file that include the reference to the posts and eventually the latest post list.

All the infomation from post, list of post, contacts, ecc. are imported into the DOM by some small JS scripts: 

$(function () {
  var includes = $('[data-include]')
  $.each(includes, function () {
    var file = $(this).data('include')
    $(this).load(file)
  })
})
Include in the html looks like that:
<‍div data-include_in="2021/08/25/new_post_overview.html"><‍/div>
